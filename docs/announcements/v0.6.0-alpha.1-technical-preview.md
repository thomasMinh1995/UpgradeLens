# DepVerdict v0.6.0-alpha.1 Technical Preview

Checking whether a registry has a newer dependency version is not the same as
deciding whether a project should upgrade. DepVerdict explores the harder
questions: should this occurrence upgrade, why, what is the risk, and what
evidence or human review is still missing?

The `0.6.0-alpha.1` Public Technical Preview provides a decision-first CLI with
deterministic policy, fail-closed handling of missing or unsupported evidence,
coverage-aware JavaScript/TypeScript impact analysis, and an optional
evidence-bounded handoff for human review.

Install the preview explicitly:

```sh
npm install -g @thomasminh1995/depverdict@preview
```

Try a zero-secret offline pass in a disposable project:

```sh
depverdict analyze . --offline --fail-on-incomplete
```

The current strengths are npm projects and workspaces, npm lockfile baselines,
and JavaScript/TypeScript source usage. Python discovery and PyPI research have
bounded support, while pnpm/Yarn installed baselines, Python environment
baselines, and non-JavaScript source analysis remain limited or unsupported.
Missing coverage never becomes proof that an upgrade is safe or that a
dependency is unused.

We would especially value feedback on:

- whether Upgrade Decisions are understandable and actionable;
- `INVESTIGATE` calibration and installed-version accuracy;
- duplicate-occurrence selector guidance;
- whether coverage limitations prevent false confidence;
- completion states and strict CI exit behavior;
- whether the human-reviewed handoff reduces repeated research.

This is an Alpha/Technical Preview, not a production-readiness or safety claim.
DepVerdict does not autonomously modify source, install dependencies, run
suggested commands, approve a migration, or provide universal ecosystem
coverage. Every target and proposed action still requires authoritative human
review.

The npm registry currently also exposes this first published version through
`latest`. That registry-imposed limitation does not change its Alpha status.
Install and evaluate it explicitly with `@preview`.

This is a reusable draft. It has not been published as a community
announcement.
